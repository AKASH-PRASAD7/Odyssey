# Cheat Sheet: Building Supervised Learning Models

## Comparing Common Supervised Learning Models

| Model Name                                       | Description                                                                                                                                                                                                                                                                                                                                                                                                                | Code Syntax                                                                                                                                                                               |
| ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **One vs One classifier (Logistic Regression)**  | **Purpose:** Trains one classifier for each pair of classes.<br><br>**Key hyperparameters:** `estimator` – Base classifier (e.g., Logistic Regression).<br><br>**Pros:** Works well for small datasets.<br><br>**Cons:** Computationally expensive for large datasets.<br><br>**Common applications:** Multiclass classification with a relatively small number of classes.                                                | 1. `from sklearn.multiclass import OneVsOneClassifier` <br><br> 2. `from sklearn.linear_model import LogisticRegression` <br><br> 3. `model = OneVsOneClassifier(LogisticRegression())`   |
| **One vs Rest classifier (Logistic Regression)** | **Purpose:** Trains one classifier per class, where each classifier distinguishes one class from all others.<br><br>**Key hyperparameters:** `estimator` – Base classifier, `multi_class='ovr'`.<br><br>**Pros:** Simpler and more scalable than One vs One.<br><br>**Cons:** Less accurate for highly imbalanced classes.<br><br>**Common applications:** Image classification and other multiclass classification tasks. | 1. `from sklearn.multiclass import OneVsRestClassifier` <br><br> 2. `from sklearn.linear_model import LogisticRegression` <br><br> 3. `model = OneVsRestClassifier(LogisticRegression())` |
| **Decision tree classifier**                     | **Purpose:** Splits data into smaller subsets based on feature values to classify data.<br><br>**Key hyperparameters:** `max_depth` – Maximum depth of the tree.<br><br>**Pros:** Easy to interpret and visualize.<br><br>**Cons:** Prone to overfitting if not pruned properly.<br><br>**Common applications:** Credit risk assessment and classification problems.                                                       | 1. `from sklearn.tree import DecisionTreeClassifier` <br><br> 2. `model = DecisionTreeClassifier(max_depth=5)`                                                                            |
| **Decision tree regressor**                      | **Purpose:** Predicts continuous values using tree-based splits.<br><br>**Key hyperparameters:** `max_depth` – Maximum depth of the tree.<br><br>**Pros:** Handles nonlinear data and easy to interpret.<br><br>**Cons:** Can overfit noisy data.<br><br>**Common applications:** Predicting housing prices and regression tasks.                                                                                          | 1. `from sklearn.tree import DecisionTreeRegressor` <br><br> 2. `model = DecisionTreeRegressor(max_depth=5)`                                                                              |
| **Linear SVM classifier**                        | **Purpose:** Finds the optimal hyperplane that maximizes the margin between classes.<br><br>**Key hyperparameters:** `C`, `kernel`, `gamma`.<br><br>**Pros:** Effective for high-dimensional datasets.<br><br>**Cons:** Not suitable for nonlinear problems without kernels.<br><br>**Common applications:** Text classification and image recognition.                                                                    | 1. `from sklearn.svm import SVC` <br><br> 2. `model = SVC(kernel='linear', C=1.0)`                                                                                                        |
| **K-Nearest Neighbors (KNN) classifier**         | **Purpose:** Predicts labels based on the majority class among nearest neighbors.<br><br>**Key hyperparameters:** `n_neighbors`, `weights`, `algorithm`.<br><br>**Pros:** Simple and effective for small datasets.<br><br>**Cons:** Computationally expensive for large datasets.<br><br>**Common applications:** Recommendation systems and image recognition.                                                            | 1. `from sklearn.neighbors import KNeighborsClassifier` <br><br> 2. `model = KNeighborsClassifier(n_neighbors=5, weights='uniform')`                                                      |
| **Random Forest regressor**                      | **Purpose:** Uses multiple decision trees to improve prediction accuracy and reduce overfitting.<br><br>**Key hyperparameters:** `n_estimators`, `max_depth`.<br><br>**Pros:** Less prone to overfitting than single decision trees.<br><br>**Cons:** Increased complexity with more trees.<br><br>**Common applications:** Sales forecasting and stock price prediction.                                                  | 1. `from sklearn.ensemble import RandomForestRegressor` <br><br> 2. `model = RandomForestRegressor(n_estimators=100, max_depth=5)`                                                        |
| **XGBoost regressor**                            | **Purpose:** Sequentially builds trees to correct previous prediction errors using gradient boosting.<br><br>**Key hyperparameters:** `n_estimators`, `learning_rate`, `max_depth`.<br><br>**Pros:** High accuracy and excellent performance on large datasets.<br><br>**Cons:** Computationally intensive and requires tuning.<br><br>**Common applications:** Predictive modeling and Kaggle competitions.               | 1. `import xgboost as xgb` <br><br> 2. `model = xgb.XGBRegressor(n_estimators=100, learning_rate=0.1, max_depth=5)`                                                                       |

# Cheat Sheet: Common Utility Functions in Supervised Learning

| Function Name             | Description                                                                                                                                                       | Code Syntax                                                                                                                                                                   |
| ------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **OneHotEncoder**         | **Purpose:** Converts categorical variables into one-hot encoded vectors.<br><br>**Common applications:** Preparing categorical data for machine learning models. | 1. `from sklearn.preprocessing import OneHotEncoder` <br><br> 2. `encoder = OneHotEncoder(sparse=False)` <br><br> 3. `encoded_data = encoder.fit_transform(categorical_data)` |
| **accuracy_score**        | **Purpose:** Measures classification accuracy by comparing predicted and true labels.<br><br>**Common applications:** Evaluating classification models.           | 1. `from sklearn.metrics import accuracy_score` <br><br> 2. `accuracy = accuracy_score(y_true, y_pred)`                                                                       |
| **LabelEncoder**          | **Purpose:** Converts target labels into numeric form.<br><br>**Common applications:** Encoding labels for machine learning algorithms.                           | 1. `from sklearn.preprocessing import LabelEncoder` <br><br> 2. `encoder = LabelEncoder()` <br><br> 3. `encoded_labels = encoder.fit_transform(labels)`                       |
| **plot_tree**             | **Purpose:** Visualizes a trained decision tree.<br><br>**Common applications:** Understanding and interpreting decision tree models.                             | 1. `from sklearn.tree import plot_tree` <br><br> 2. `plot_tree(model, max_depth=3, filled=True)`                                                                              |
| **normalize**             | **Purpose:** Scales features to unit norm.<br><br>**Common applications:** Feature preprocessing for distance-based models like KNN and SVM.                      | 1. `from sklearn.preprocessing import normalize` <br><br> 2. `normalized_data = normalize(data, norm='l2')`                                                                   |
| **compute_sample_weight** | **Purpose:** Computes sample weights for imbalanced datasets.<br><br>**Common applications:** Handling class imbalance during model training.                     | 1. `from sklearn.utils.class_weight import compute_sample_weight` <br><br> 2. `weights = compute_sample_weight(class_weight='balanced', y=y)`                                 |
| **roc_auc_score**         | **Purpose:** Computes the Area Under the ROC Curve (AUC-ROC).<br><br>**Common applications:** Evaluating binary classification models.                            | 1. `from sklearn.metrics import roc_auc_score` <br><br> 2. `auc = roc_auc_score(y_true, y_score)`                                                                             |
